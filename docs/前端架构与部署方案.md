# 技术栈
核心技术选型
类别	选型	理由
框架	React 18 + TypeScript	AntD v5基于React；.tsx要求已隐含TypeScript
构建工具	Vite 5	启动快、配置简单，适合多应用工程
UI组件库	Ant Design v5	需求明确指定
路由	React Router v6	左侧多模块切换布局需要
HTTP请求	Axios（封装统一拦截器）	统一鉴权、错误处理
状态管理	Zustand	轻量，400人规模无复杂全局态，避免Redux过重
样式方案	CSS Modules（xxx.module.css）	满足"样式分离到同名.css、.tsx无样式代码"要求

# 单应用前端部署方案（Docker）
以 csun_server-app 为例，多阶段构建：构建期用Node编译，运行期仅保留Nginx+静态产物，镜像小、安全。
1. Dockerfile（放置于 frontend/）
# ---------- 阶段1：构建 ----------
FROM node:20-alpine AS builder
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci --registry=https://registry.npmmirror.com
COPY . .
RUN npm run build

# ---------- 阶段2：运行 ----------
FROM nginx:1.27-alpine
# 拷贝构建产物
COPY --from=builder /app/dist /usr/share/nginx/html
# 拷贝Nginx配置
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]

2. nginx.conf（同目录）
server {
    listen 80;
    server_name _;

    root /usr/share/nginx/html;
    index index.html;

    # SPA路由回退（React Router history模式）
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 反向代理到统一Gin后端
    location /api/ {
        proxy_pass http://backend:8080/;   # backend为后端容器名/服务名
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|svg|woff2?)$ {
        expires 7d;
        add_header Cache-Control "public, immutable";
    }
}

3. 构建与运行
# 在 frontend 目录下
docker build -t csun_server-app:1.0.0 .

# 运行（端口8081映射到容器80）
docker run -d --name csun_server-app --network csun_net -p 8081:80 csun_server-app:1.0.0

4. 多应用编排（docker-compose.yml，置于项目根）
各应用独立容器、统一后端、统一网络，满足"互不影响"：
services:
  backend:
    image: csun_server-backend:1.0.0
    ports: ["8080:8080"]
    networks: [csun_net]

  csun_server-app:
    build: ./frontend
    ports: ["8081:80"]
    depends_on: [backend]
    networks: [csun_net]

  # 未来新增应用只需在此追加一个service
networks:
  csun_net:
